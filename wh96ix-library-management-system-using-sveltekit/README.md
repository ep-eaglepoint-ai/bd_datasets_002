## Requirements
- Admin can add, update, delete, and view books.
- System must store book details (title, author, ISBN, category, total copies, available copies).
- Borrowers can search and view available books.
- Borrowers can borrow and return books.
- System must assign and track due dates for borrowed books.
- System must identify overdue books.
- Users can view borrowing history.
- Authentication and role-based access (Admin / Borrower).
- Built using SvelteKit framework.
- Responsive and user-friendly UI.
- Secure authentication and authorization.
- Database integration for persistent storage.
